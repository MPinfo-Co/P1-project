# P1 缺口與孤立文件掃描任務

> [← 回到總導覽](../README.md)

## 任務目標
掃描各 Repo 的檔案，找出未被任何文件覆蓋的重要主題（缺口）與未被索引的孤立文件。

## 前置條件
本任務假設已執行完 [AI-review-prompt.md](AI-review-prompt.md)，可直接利用該次對話中已讀取的文件內容。

## 限制
- 本次任務為唯讀分析，不修改任何檔案

## 執行步驟

### Step 1 — 掃描各 Repo 檔案
掃描各 Repo 的 `.md` 檔案，排除以下目錄：
- `node_modules/`、`.git/`、`__pycache__/`、`.pytest_cache/` 等非專案目錄
- `P1-analysis/references/`（全域參考文件，不受索引追蹤）
- `P1-design/TestPlan/`（Delta Record，以 Issue 為單位產生）

### Step 2 — 比對索引
將掃描結果與各文件中的目錄結構描述、索引表格、連結進行比對，識別：
- **孤立文件**：存在於 Repo 中但未被任何文件索引到的檔案
- **缺口**：重要主題未被任何文件覆蓋

---

## 報告格式
> 將以下報告內容寫入 `docs/AI-review-gap-report.md`。
> - 每次執行為獨立區塊，以 `## YYYY-MM-DD` 標題分隔
> - 最新一次插入檔案最上方（既有內容往下推）
> - 僅保留最近 3 次，超過的區塊刪除（完整歷史由 git 追溯）
